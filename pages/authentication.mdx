<Tip>
  **Token de Acesso (JWT)**: Todas as requisições à API da V4Pay devem conter um token JWT válido no header `Authorization`. Este token identifica e autentica o usuário da requisição.
</Tip> 

## <Icon icon="lock" type="solid"/>  Gerenciando chaves de API

Gerencie suas chaves de API diretamente em nossa plataforma. Você pode:
- Listar todas as chaves ativas
- Criar novas chaves
- Revogar chaves existentes

<Card title="Ambientes e Chaves de API" icon="book" horizontal>
  Todas as requisições são enviadas para o mesmo endpoint (`https://api.v4pay.com`), mas o ambiente é determinado pela chave de API utilizada:
  - Chaves criadas em **dev mode** processam transações no ambiente de testes
  - Chaves criadas em **produção** processam transações reais
  
  Saiba mais sobre o ambiente de desenvolvimento <a href="./devmode">aqui</a>.
</Card>

<Card title="Erro de Autenticação" icon="warning" horizontal>
  A API retornará o código HTTP `401` quando:
  - O token JWT não for fornecido no header
  - O token for inválido ou expirado
</Card>

## <Icon icon="key" type="solid"/>  Criando chaves de API

<Card title="Boas Práticas de Segurança" icon="light-emergency-on" horizontal>
  * Armazene suas chaves em variáveis de ambiente ou gerenciadores de segredos
  * Nunca compartilhe suas chaves de API
  * A V4Pay nunca solicitará suas chaves
  * Revogue imediatamente qualquer chave comprometida
</Card>

Após criar sua chave, você pode começar a integrar com nossa API. Lembre-se de incluir o token JWT no header `Authorization`:

```bash
curl -X POST https://api.v4pay.com/v1/payments \
  -H "Authorization: Bearer token" \
  -H "Content-Type: application/json" \
  -d '{"amount": 1000}'
```

<Card title="Nota Importante" icon="info" horizontal>
  O mesmo endpoint (`https://api.v4pay.com`) é usado tanto para o ambiente de desenvolvimento quanto para produção.
</Card>
