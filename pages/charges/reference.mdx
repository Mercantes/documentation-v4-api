## Introdução

O termo "cobrança" é genérico. Ele representa um portal de um fluxo de pagamento onde você pode cobrar seu cliente e ele fazer todo o processo de pagamento sem nenhuma interrupção.

Atualmente uma cobrança pode ser de duas formas:
- `ONE_TIME`: cobrança única, exige dados do cliente (`customerId` ou `customer`)
- `MULTIPLE_PAYMENTS`: pode ser paga várias vezes por diferentes pessoas via checkout aberto.

## Endpoints

### Criar cobrança

<Endpoint
  method="POST"
  path="/charges"
  description="Cria uma nova cobrança associada a um cliente e usuário autenticado."
  requestBody={{
    customer_id: 'ID do cliente (UUID)',
    user_id: 'ID do usuário da plataforma (UUID)',
    amount: 120,
    description: 'Assinatura Premium',
    expires_at: '2025-12-31T23:59:59Z'
  }}
  responses={{
    201: {
      description: 'Cobrança criada com sucesso'
    },
    400: {
      description: 'Campos obrigatórios ausentes ou inválidos'
    },
    401: {
      description: 'Token JWT ausente ou inválido'
    },
    500: {
      description: 'Erro interno no servidor'
    }
  }}
/>

### Listar cobranças

<Endpoint
  method="GET"
  path="/charges"
  description="Lista todas as cobranças registradas vinculadas ao usuário autenticado."
  responses={{
    200: {
      description: 'Lista de cobranças retornada com sucesso'
    },
    401: {
      description: 'Token JWT ausente ou inválido'
    },
    500: {
      description: 'Erro ao buscar as cobranças'
    }
  }}
/>

### Checar status da cobrança

<Endpoint
  method="GET"
  path="/charges/{id}/status"
  description="Consulta o status de uma cobrança específica por ID."
  parameters={[
    {
      name: 'id',
      in: 'path',
      required: true,
      description: 'ID da cobrança a ser consultada'
    }
  ]}
  responses={{
    200: {
      description: 'Status da cobrança retornado com sucesso'
    },
    401: {
      description: 'Token JWT ausente ou inválido'
    },
    404: {
      description: 'Cobrança não encontrada'
    },
    500: {
      description: 'Erro ao buscar status da cobrança'
    }
  }}
/>

## <Icon icon="folder-tree" type="solid"/> Estrutura

Uma cobrança é representada em nossa API pela seguinte estrutura:

```json json
{
  "id": "bill_uA0M0xwg5R4mSyr0n2PjHQXY",
  "frequency": "ONE_TIME",
  "url": "https://v4pay.com/pay/bill_uA0M0xwg5R4mSyr0n2PjHQXY",
  "amount": 4000,
  "status": "PAID",
  "devMode": true,
  "methods": ["PIX"],
  "products": [
    {
      "id": "prod_dNFbdDjfpaegmzBWWdNM2Huw",
      "externalId": "prod-1234",
      "quantity": 1
    }
  ],
  "customer": {
    "id": "cust_aebxkhDZNaMmJeKsy0AHS0FQ",
    "metadata": {
      "name": "Test Customer",
      "cellphone": "11999999999",
      "taxId": "12345678900",
      "email": "test@example.com"
    }
  },
  "metadata": {
    "fee": 100,
    "returnUrl": "https://example.com/billing",
    "completionUrl": "https://example.com/completion"
  },
  "nextBilling": null,
  "allowCoupons": false,
  "coupons": [],
  "createdAt": "2024-12-06T18:56:15.538Z",
  "updatedAt": "2024-12-06T18:56:15.538Z"
}
```

        <Info >
            |                      |                                                                                                      |
            | ---------------------| ---------------------------------------------------------------------------------------------------- |
            | `ONE_TIME`           | **Cobrança que aceita um único pagamento. É necessário enviar os dados do cliente**                  |
            | `MULTIPLE_PAYMENTS`  | **Cobrança em modo checkout, aceita vários pagamentos. Não é necessário enviar os dados do cliente.**|
        </Info>
    </Accordion>
     <Accordion title="url:">
       ```json json {2}

        }
        ```
        `url` : <u> string. </u> <br />
        URL para seu cliente executar o pagamento da cobrança

       <Accordion title="status:">

        `status` : <u> string. </u> <br />

        <Info >
            |  |                            |
            | -----------| ------------------------------------- |
            | `PENDING`  | **A cobrança está com o pagamento pendente**                    |
            | `EXPIRED`  | **O tempo limite de pagamento foi excedido**                          |
            | `CANCELLED`| **A cobrança foi cancelada por você** |
            | `PAID`     | **A cobrança foi paga com sucesso pelo cliente** |
            | `REFUNDED` | **O valor foi devolvido ao cliente** |
        </Info>
    </Accordion>
