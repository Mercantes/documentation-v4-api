Um cliente é seu usuário final, aquele que você vai cobrar e pagar o seu produto.

## <Icon icon="folder-tree" />  Estrutura

Um cliente é representado em nossa API pela seguinte estrutura:

```json
{
  "id": "cust_9a930226-d8b2-44",
  "user_id": "user_xpto",
  "name": "Vinícius",
  "email": "vinicius@example.com",
  "cpf": "22898355879",
  "phone": "11999999999",
  "created_at": "2025-07-09T16:01:54.018457+00:00",
  "updated_at": "2025-07-09T16:01:54.018457+00:00"
}
```

## Atributos

### id:

```json
{
  "id": "cust_9a930226-d8b2-44"
}
```

`id`: <u>string</u>\
 ID público do cliente

### user_id:

```json
"user_id": "user_xpto"
```

`user_id`: <u>string</u>\
 Identificador único do usuário dono do cliente

### name:

`name`: <u>string</u>\
 Nome do cliente

### email:

`email`: <u>string</u>\
 E-mail do cliente (obrigatório)

### cpf:

`cpf`: <u>string</u>\
 CPF do cliente

### phone:

`phone`: <u>string</u>\
 Telefone do cliente

### created_at / updated_at:

`created_at`, `updated_at`: <u>string (ISO date)</u>\
 Datas de criação e atualização do cliente

<Tip>
  Não é possível criar um cliente com CPF/CNPJ inválido e caso já exista um cliente com esse CPF/CNPJ, a API não criará o cliente novo mas retornará o cliente já existente
</Tip>
